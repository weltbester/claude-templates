# pm-requirements

Gather and analyze product requirements

## Task
Facilitate product management and requirements analysis:
- Feature requirements gathering
- User story creation
- Acceptance criteria definition
- Technical feasibility analysis
- Effort estimation
- Dependency mapping
- Risk identification
- Success metrics definition

## Context
$ARGUMENTS

## Process
1. **Discovery**: Understand the problem and user needs
2. **Analysis**: Break down requirements into actionable items
3. **Specification**: Document features with clear acceptance criteria
4. **Planning**: Identify dependencies, risks, and milestones
5. **Validation**: Ensure requirements are testable and measurable

## Output Format
### Feature Overview
- **Objective**: What problem does this solve?
- **User Stories**: As a [user], I want [goal] so that [benefit]
- **Acceptance Criteria**: Specific, testable conditions

### Technical Considerations
- Implementation approach
- Dependencies and integrations
- Risks and mitigations
- Estimated complexity

### Success Metrics
- How will we measure success?
- Key performance indicators
