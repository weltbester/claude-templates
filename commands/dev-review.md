# dev-review

Perform a comprehensive development review

## Task
Conduct a thorough development review covering:
- Code quality and best practices
- Architecture and design patterns
- Performance considerations
- Security vulnerabilities
- Type safety and error handling
- Code documentation and maintainability
- Test coverage and quality
- Dependency usage and updates

## Context
$ARGUMENTS

## Output
Provide a structured review with:
1. **Strengths**: What's well-implemented
2. **Issues**: Problems found (categorized by severity)
3. **Recommendations**: Specific improvements with examples
4. **Action Items**: Prioritized list of next steps
